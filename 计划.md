## 结构说明

- **一篇笔记讲透一个名词** — 如果子概念过多内容大，升级为文件夹递归拆分
- **三步叙事**：历史动机 → 是什么 → 怎么用
- **递归解释**：解释 A 时出现术语 B，当场解释 B，确保读任何笔记都不被生词卡住
- **跨域重复处理**：同一名词在多个领域出现时，在**核心领域**完整走三步（历史动机 → 是什么 → 怎么用）讲透；其他领域只讲"本视角下的应用"
- **先横向对比再纵向挖深**：讲一个名词时，在"是什么"阶段先列出几个同类主流工具/概念并做简短对比，让读者知道品类全貌，然后再纵向深挖当前主角。
- 全部使用 `.ipynb`，代码放 code cell 并用 cell 级 metadata 指定语言
