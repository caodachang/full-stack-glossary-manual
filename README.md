## 笔记约定

一篇笔记讲透一个名词，要讲的尽可能详细，如果某个名词下有太多其他名词总内容太多可以当作文件夹再往下写。每个名词按三步组织：

> **为什么需要它 → 是什么 → 怎么样**

### 递归解释

解释名词 A 的过程中，如果出现了名词 B，当场解释 B 是什么。解释 B 时如果又出现了 C，继续解释 C。确保读任何一个名词的笔记时，不会因为中间出现生词而卡住。

### 笔记格式

全部使用 `.ipynb`。代码块利用 Jupyter 的语法高亮即可，不需要可运行，只需要看起来清晰易读。

### 笔记互链

笔记之间用相对路径互相引用，例如：

> 想深入 Nginx 的 HTTP 层 → [05-通信/HTTP概述.ipynb](05-通信/HTTP概述.ipynb)
> 前端调用 API 的认证细节 → [11-安全/认证与授权/JWT.ipynb](11-安全/认证与授权/JWT.ipynb)
